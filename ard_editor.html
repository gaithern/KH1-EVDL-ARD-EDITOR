ARD ENTITY FIELD LEGEND

Field layout verified from live memory and cross-referenced with the EVDL decompiler.
Entry size: 0x78 bytes. All floats are little-endian IEEE 754. All integers are little-endian.

⬡ IDENTITY (+0x00 – +0x07)
OFFSET TYPE FIELD DESCRIPTION
+0x00u16 slot (flags) Lo-word of EVDL identity dword. Sequential index used by EVDL scripts to look up this entity. Must be unique within the section. Sora=0x0000, Donald=0x0001, Goofy=0x0002.
+0x02u16 v1 (group) Hi-word of EVDL identity dword. Entity group — defines which class of object this is. Combined with slot: EVDL push value = v1 × 0x10000 + slot. Known: 3=players, 4=props, 9=boxes. Do not change unless you know the correct group.
+0x04u32 push_cond Condition ID read at runtime by the push_cond (0x15) EVDL opcode. The engine checks this value against world/room flags to gate script execution for this entity. Copy from an entity of the same type — do not invent values.
+0x08u32 padding Always 0x00000000.
+0x0Cu32 RR (reserved) Written by the game engine at runtime. Not meaningful in the file — do not edit.

ENTITY STRING (+0x68 – +0x77)
+0x68char[16]ES — Entity string. Null-terminated C-string, padded to 16 bytes. Used as the entity name in EVDL scripts and debug display. Max 15 characters + null.
NOTE ON push_cond AND EVDL OPCODE 0x15
The push_cond (0x15) opcode in the EVDL VM reads the push_cond field from the entity's runtime slot before evaluating world/room flags. It is not a generic wide-immediate push — it gates script execution on game state. This means two entities of the same type should share the same push_cond value. Always copy it from an existing entity of the same class.
Sources: ARD Documentation by Some1fromthedark · EVDL decompiler (handler verified @ 0x1404E5DB0) · live memory analysis
